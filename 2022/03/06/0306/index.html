前端代码规范，学会了，你也能写出诗一样的代码！

发表于 2022-03-06 更新于 2022-03-15

前端代码规范

如果你有去过大厂，或者说有过大厂相关的项目经历，相信你一定对代码规范有所了解。而作为大厂的前端，有一项是必须要做的事情，就是要遵守代码规范，并写出像诗一样的代码😁
那么为什么大厂会这么会这么重视代码规范呢，其实答案很简单。通常代码的项目都很大，很多人并行开发，然而每个人的写代码的习惯都不一样，例如有的人习惯写注释，有的人却没有写注释的习惯，所以为了统一大家的代码风格，
为了便于日后的代码维护，所以制定一个统一的规范，以此来进行约束，代码规范自然就应运而生了。那么本片文章就结合自己的工作经历，来给大家介绍一下业内比较好的几个代码规范，准备好了么，Let’s go! 😏

腾讯代码规范

包含内容也挺多的：
PC端专题：快速上手、文件目录、页面头部、通用title、通用foot、统计代码、兼容测试
移动端专题：快速上手、文件目录、页面头部、REM布局、通用foot、统计代码、分享组件、兼容要求
双端官网：快速上手、页面跳转

京东凹凸实验室前端代码规范

HTML规范:
基于W3C、苹果开发者等官方文档，并结合团队日常业务需求以及团队在日常开发过程中总结提炼出的经验而约定。
图片规范:
了解各种图片格式特性，根据特性制定图片规范，包括但不限于图片的质量约定、图片引入方式、图片合并处理等。
CSS 规范:
统一团队 CSS 代码书写和 SASS 预编译语言的语法风格，提供常用媒体查询语句和浏览器私有属性引用，并从业务层面统一规范常用模块的引用。
命名规范:
从 “目录命名”、“图片命名”、“ClassName” 命名等层面约定规范团队的命名习惯，增强团队代码的可读性。
JavaScript 规范:
统一团队的 JS 语法风格和书写习惯，减少程序出错的概率，其中也包含了 ES6 的语法规范和最佳实践。

百度前端代码规范

百度一直是一个相信代码可以改变世界的公司，也是中国大厂里面第一个把代码规范做成标准化的公司，所以他们的代码规范一直被模仿，值得学习！

vue编码规范

作为一款国民前端框架，如果你所在的项目正好在使用vue,那么vue特有的代码风格指南可以关注下！

ESLint

目前绝大多数前端项目都会用到的 可组装的 JavaScrip t和 JSX 检查工具。
在我看来 ESLint 的核心可能就是其中包含的各种规则，这些规则大多为众多开发者经验的结晶：
*有的可以帮我们避免错误；
*有的可以帮我们写出最佳实践的代码；
*有的可以帮我们规范变量的使用方式；
*有的可以帮我们规范代码格式；
*有的可以帮我们更合适的使用新的语法.

以上就是本文的全部内容，如果你目前所在的项目还没有建立完善的代码规范而恰巧又需要引入代码规范，那么本文所罗列的这些业界代码规范可以供你参考！
